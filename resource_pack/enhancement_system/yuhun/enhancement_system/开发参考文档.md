## 核心概念

### 路线（Route）

路线是评估御魂的核心概念。每个御魂配置可以定义多条路线，每条路线代表一种使用场景。

```python
@dataclass
class Route:
    路线名称: str                    # "常规输出"、"极限速度"等
    有效属性: List[str]              # ["暴击", "暴击伤害", "速度", "攻击加成"]
    有效属性权重: List[float]        # [1, 1, 1, 1] 或 [1, 1, 0.5, 0.5]
```

**评分公式**：
```
评分 = Σ(等效强化次数 × 权重) for 有效属性
```

**Top N 判断**：
- 单属性路线（如极限速度）：Top 3
- 多属性路线（如常规输出）：Top 10

**最低分数阈值**：
- 单属性路线：4.5
- 多属性路线：5.4

**最终阈值** = max(Top N 阈值, 最低分数)

## 模块说明

### 1. models.py - 数据模型

```python
@dataclass
class SubAttribute:
    类型: str              # "暴击"、"速度"等
    数值: str              # 原始字符串 "3.0%"
    强化次数: int          # 该条被强化的次数
    等效强化次数: float    # 归一化质量指标 = 数值 / 单次最大值

@dataclass
class MainAttribute:
    类型: str
    数值: str

@dataclass
class Equip:
    位置: int              # 0-5 对应 1-6号位
    类型: str              # 套装名称，如"破势"
    品质: int              # 6=SSR
    等级: int              # 0/3/6/9/12/15
    主属性: MainAttribute
    副属性: List[SubAttribute]

    @property

    @property
    def 是否满级(self) -> bool

@dataclass
class Route:
    路线名称: str
    有效属性: List[str]
    有效属性权重: List[float]

@dataclass
class EquipConfig:

    def get_routes(self, 位置: int, 主属性类型: str) -> List[Route]
```

### 3. scoring.py - 评分计算

```python
def calculate_equivalent_enhancement(attr_type: str, value: float) -> float:
    """计算等效强化次数"""
    # 等效强化次数 = 实际数值 / 单次最大值
    # 例如：速度 3.0 / 3.0 = 1.0次

class Scorer:
    """评分器"""
    def __init__(self, compatible_attrs: List[str], weights: List[float])
    def score(self, equip: Equip) -> float
    # 评分 = Σ(等效强化次数 × 权重) for 有效属性

def create_scorer_for_route(route: Route) -> Scorer:
    """根据路线创建评分器"""
    return Scorer(route.有效属性, route.有效属性权重)
```

| 防御 | 3 | 4 | 数值 |
| 防御加成 | 2.4% | 3.0% | 百分比 |
| 生命 | 81 | 114 | 数值 |
| 生命加成 | 2.4% | 3.0% | 百分比 |
| 速度 | 2.4 | 3.0 | 数值 |
| 暴击 | 2.4% | 3.0% | 百分比 |
| 暴击伤害 | 3.2% | 4.0% | 百分比 |

### 4. enhancement_calculator.py - 概率计算

```python
@dataclass
class EnhanceState:
    """强化状态"""
    sub_attrs: Tuple[Tuple[str, int], ...]  # ((类型, 强化次数), ...)
    
class EnhancementCalculator:
    def __init__(self, scorer: Scorer)
    
    def calculate_probability_distribution(
        self, equip: Equip, target_level: int
    ) -> Dict[EnhanceState, float]
    # 计算所有可能状态及其概率
    
    def calculate_score_distribution(
        self, equip: Equip, target_level: int
    ) -> List[Tuple[float, float]]
    # 计算评分分布 [(评分, 概率), ...]
    
    def estimate_expected_score(
        self, equip: Equip, target_level: int
    ) -> float
    # 计算期望评分
    
    def calculate_top_n_probability(
        self, equip: Equip, target_level: int, threshold: float
    ) -> float
    # 计算达到阈值的概率
```

**强化规则**：
- 0级御魂：+3/+6/+9/+12/+15 各强化一次
- 3腿御魂：+3时新增第4条副属性（11种可能，各1/11概率）
- 4腿御魂：每次强化随机选择一条副属性（各1/4概率）

### 6. decision_engine.py - 决策引擎

```python
@dataclass
class Decision:
    action: str           # "继续强化" / "弃置"
    reason: str           # 决策理由
    probability: float    # 达到阈值的概率
    threshold: float      # 阈值评分
    current_score: float  # 当前评分
    top_n: int           # Top N值

class DecisionEngine:
    def __init__(self, pool: EquipPool, loader: DataLoader)
    
    def evaluate(self, equip: Equip, config: EquipConfig) -> Decision
    # 评估御魂，返回决策
```

**决策逻辑**：

1. 获取该御魂的所有路线配置
2. 对每条路线：
   - 计算当前评分
   - 获取 Top N 阈值（含最低分数）
   - 计算达到阈值的概率
3. 选择概率最高的路线
4. 判断：
   - 概率 ≥ 阶段阈值 → 继续强化
   - 概率 < 阶段阈值 → 弃置

**阶段阈值** (stage_thresholds)：
```json
{
  "0": 0.30,   // +0 御魂需要 ≥30% 概率
  "3": 0.40,   // +3 御魂需要 ≥40% 概率
  "6": 0.50,
  "9": 0.60,
  "12": 0.70
}
```

## 御魂配置格式

御魂配置文件位于 `御魂数据/` 目录，每个御魂一个 JSON 文件。

```json
{
  "御魂名称": "破势",
  "两件套属性": "暴击",
  "两件套数值": "15%",
  "四件套效果": "...",
  "位置配置": {
    "1号位": {
      "固定主属性": "攻击",
      "路线": [
        {
          "路线名称": "常规输出",
          "有效属性": ["暴击", "暴击伤害", "速度", "攻击加成"],
          "有效属性权重": [1, 1, 1, 1]
        }
      ]
    },
    "2号位": {
      "可选主属性": ["攻击加成", "防御", "生命", "速度", "效果命中", "效果抵抗"],
      "速度": {
        "路线": [
          {
            "路线名称": "常规输出",
            "有效属性": ["暴击", "暴击伤害", "速度", "攻击加成"],
            "有效属性权重": [1, 1, 1, 1]
          },
          {
            "路线名称": "极限速度",
            "有效属性": ["速度"],
            "有效属性权重": [1]
          }
        ]
      }
    }
  }
}
```

**路线配置说明**：
- `路线名称`：路线的描述性名称
- `有效属性`：该路线关注的副属性列表
- `有效属性权重`：对应属性的权重，通常为 1，可以设置不同权重来调整重要性

## 使用示例

### 评估单个御魂

```python
from core.data_loader import DataLoader
from core.decision_engine import DecisionEngine

# 加载数据
loader = DataLoader()
pool = loader.load_equip_pool()
engine = DecisionEngine(pool, loader)

# 加载御魂配置
config = loader.load_equip_config("破势")

# 评估御魂
decision = engine.evaluate(equip, config)
print(f"决策: {decision.action}")
print(f"理由: {decision.reason}")
print(f"概率: {decision.probability:.1%}")
```

### 创建评分器

```python
from core.models import Route
from core.scoring import create_scorer_for_route

# 定义路线
route = Route(
    路线名称="常规输出",
    有效属性=["暴击", "暴击伤害", "速度", "攻击加成"],
    有效属性权重=[1, 1, 1, 1]
)

# 创建评分器
scorer = create_scorer_for_route(route)

# 评分
score = scorer.score(equip)
```

### 计算概率

```python
from core.enhancement_calculator import EnhancementCalculator

calc = EnhancementCalculator(scorer)

# 计算达到阈值的概率
prob = calc.calculate_top_n_probability(equip, target_level=15, threshold=5.4)
print(f"达到阈值的概率: {prob:.1%}")
```

## 已知限制和未来改进

### 1. 主属性价值评估

**问题**：当前评分只考虑副属性，完全忽略主属性的价值。

**影响**：
- 2/4/6号位的主属性选择对御魂价值影响很大
- 某些主属性组合可能比副属性更重要

**建议方案**：
- 在配置中为每个位置的主属性设置基础分
- 评分时加入主属性分数

**优先级**：低

### 2. 套装效果价值评估

**问题**：当前只考虑单个御魂的属性，未考虑凑套装的价值。

**影响**：
- 某些情况下，属性稍差但能凑成四件套的御魂可能更有价值
- 跨套装对比时无法体现套装效果的差异

**建议方案**：
- 需要更高层的"御魂组合优化"模块
- 超出当前单御魂评估的范围

**优先级**：低（需要重构架构）

### 3. 动态阈值调整

**问题**：当前 `stage_thresholds` 是固定的，未考虑御魂池的实际情况。

**潜在优化**：
- 如果某个位置的御魂池很小，可以适当降低阈值
- 如果某个位置已经有很多高分御魂，可以提高阈值

**建议方案**：
- 在决策时根据当前排名池大小动态调整阈值
- 在配置中增加"最小池大小"参数

**优先级**：低

## 版本历史

### v2.0 - 路线系统重构 (2026-05)

**重大变更**：
- 路线现在直接在御魂配置 JSON 中定义
- 移除了自动路线拆分逻辑
- 简化了 `config_parser.py`（标记为废弃）
- 移除了旧的 `create_scorer()` API
- 更新了所有测试文件

**新增功能**：
- 支持属性权重配置
- 双重阈值系统（Top N + 最低分数）
- 更灵活的路线定义

**迁移指南**：
- 使用 `create_scorer_for_route(route)` 替代 `create_scorer(recommended_attrs)`
- 使用 `config.get_routes()` 获取路线，而不是 `config.get_recommended_attrs()`
- 路线对象从 `core.models.Route` 导入，而不是 `core.config_parser.Route`

### v1.0 - 初始版本

- 基础决策系统
- 自动路线拆分
- 概率计算
- Top N 排名
