## 2025-05-XX 术语统一与极限属性配置增强

### 主要改动

#### 1. 术语统一
**目的**: 统一所有配置文件和代码中的属性名称，避免混淆。

**变化**:
- 所有御魂数据JSON文件：
  - `"暴伤"` → `"暴击伤害"`
  - `"命中"` → `"效果命中"`
  - `"抵抗"` → `"效果抵抗"`

**影响文件**:

- `core/config_parser.py`
- `tests/*.py` (所有测试文件)

#### 2. 极限属性忽略主属性配置
**目的**: 允许配置哪些极限属性在排名时可以忽略主属性。

**新增配置**:
```json
"extreme_ignore_main_attr": {
  "速度": true,     // 极限速度可以跨主属性对比
  "生命": false,    // 极限生命必须主属性为生命加成
  "攻击": false,    // 极限攻击必须主属性为攻击加成
  "防御": false,    // 极限防御必须主属性为防御加成
  "命中": true,     // 极限命中可以跨主属性对比
  "抵抗": true      // 极限抵抗可以跨主属性对比
}
```

**逻辑**:
- `true`: 排名时忽略主属性，只按位置和套装筛选
- `false`: 排名时必须主属性一致

**原因**:
- 极限生命/攻击/防御：主属性必须与极限类型一致才有意义
- 极限速度/命中/抵抗：主属性可以是任意的，应该跨主属性对比找最好的

**修改文件**:
- `config/decision_config.json`: 添加 `extreme_ignore_main_attr` 配置
- `core/decision_config.py`: 加载新配置字段
- `core/ranking.py`: 使用配置决定是否忽略主属性
- `config/README.md`: 添加配置说明

- ✅ `test_integration.py`

## 2025-05-XX 极限属性配置拆分

### 主要改动

#### 1\. 极限属性Top N值独立配置

**目的**: 允许为每种极限属性单独设置不同的Top N值，提供更灵活的配置。

**变化**:

*   原配置: `"极限属性": 3` （所有极限属性共用一个值）
*   新配置: 拆分为6个独立配置

**优势**:

*   可以为不同极限属性设置不同的保留标准
*   例如：极限速度只保留第一名，极限生命保留前5名

**修改文件**:

*   `config/decision_config.json`: 拆分极限属性配置
*   `core/decision_config.py`: 添加 `御魂类型` 字段
*   `core/config_parser.py`:
    *   `calculate_top_n()` 方法支持查找具体的极限类型配置
    *   `split_routes()` 方法使用 `"极限{type}"` 作为 equip\_type

#### 2\. 添加御魂类型字段

**新增字段**:

```
"御魂类型": "破势"
```

**作用**: 标识当前配置针对的御魂类型，方便管理多个配置文件。

#### 3\. 两件套属性统一

**修改**:

*   `two_set_mapping` 中 `"攻击": "攻击"` 改为 `"攻击": "攻击加成"`
*   御魂数据目录 `攻击` 重命名为 `攻击加成`
*   所有攻击类御魂配置文件中的 `"两件套属性": "攻击"` 改为 `"攻击加成"`

**影响文件**:

*   `config/decision_config.json`
*   `D:\Software\yys\resource\yuhun\御魂数据\攻击加成\*.json` (11个文件)

### 使用方法

**示例 - 为不同极限属性设置不同标准**:

```
"top_n_values": {
  "极限速度": 1,    // 极限速度只保留最快的
  "极限生命": 5,    // 极限生命保留前5名
  "极限攻击": 3,    // 极限攻击保留前3名
  "极限防御": 3,
  "极限命中": 10,   // 极限命中标准放宽到前10名
  "极限抵抗": 10,
  "输出御魂": 10,
  "奶妈御魂": 10,
  "防御输出": 10,
  "其他": 5
}
```

### 向后兼容性

所有改动保持向后兼容，现有代码无需修改即可运行。

## 2025-05-XX 配置文件化改造

### 主要改动

#### 1\. 修复极限属性映射错误

**问题**: 极限生命/攻击/防御属性同时查看固定数值和百分比加成，导致评估不准确。

**修复**:

*   极限生命 → 只看"生命加成%"，不看"生命"数值
*   极限攻击 → 只看"攻击加成%"，不看"攻击"数值
*   极限防御 → 只看"防御加成%"，不看"防御"数值

**影响文件**:

*   `core/config_parser.py`: 修改 `EXTREME_ATTR_MAPPING`

#### 2\. 配置参数外部化

**目的**: 将硬编码的决策参数移到配置文件，方便调整而无需修改代码。

**新增文件**:

*   `config/decision_config.json`: 决策参数配置文件
*   `core/decision_config.py`: 配置加载模块
*   `config/README.md`: 配置说明文档

**配置项**:

```
{
  "stage_thresholds": {      // 各阶段概率阈值
    "3": 0.30,
    "6": 0.40,
    "9": 0.50,
    "12": 0.60
  },
  "top_n_values": {          // Top N目标值
    "极限属性": 3,
    "输出御魂": 10,
    "奶妈御魂": 10,
    "防御输出": 10,
    "其他": 5
  },
  "extreme_attr_mapping": {  // 极限属性映射
    "速度": ["速度"],
    "生命": ["生命加成"],
    "攻击": ["攻击加成"],
    "防御": ["防御加成"],
    "命中": ["效果命中"],
    "抵抗": ["效果抵抗"]
  },
  "base_compatible_attrs": { // 基础适配属性集
    "输出御魂": ["暴击", "暴击伤害", "速度", "攻击加成"],
    "奶妈御魂": ["暴击", "暴击伤害", "速度", "生命加成"],
    "防御输出": ["暴击", "暴击伤害", "速度", "防御加成"],
    "其他": ["速度", "暴击", "暴击伤害"]
  },
  "two_set_mapping": {       // 两件套属性映射
    "生命": "生命加成",
    "攻击": "攻击",
    "防御": "防御加成"
  }
}
```

**修改文件**:

*   `core/config_parser.py`: 从配置文件读取参数
*   `core/decision_engine.py`: 从配置文件读取阶段阈值
*   `core/ranking.py`: 从配置文件读取两件套映射

#### 3\. 测试更新

**修改文件**:

*   `tests/test_route_splitting.py`: 更新测试用例以匹配新的极限属性映射

### 使用方法

修改 `config/decision_config.json` 中的参数，重新运行程序即可生效。

**示例 - 调整保守程度**:

```
// 更激进（保留更多御魂）
"stage_thresholds": {
  "3": 0.20,
  "6": 0.30,
  "9": 0.40,
  "12": 0.50
}

// 更保守（只保留最好的）
"stage_thresholds": {
  "3": 0.40,
  "6": 0.50,
  "9": 0.60,
  "12": 0.70
}
```

**示例 - 调整排名目标**:

```
// 更严格的极限属性标准
"top_n_values": {
  "极限属性": 1,  // 只保留第一名
  "输出御魂": 5,
  ...
}
```
