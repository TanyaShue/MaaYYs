# 决策配置说明

## decision\_config.json

这个配置文件包含了御魂强化决策系统的所有可调参数。修改此文件后，重新运行程序即可生效，无需修改代码。

### 配置项说明

#### 0\. 御魂类型

指定当前要评估的御魂类型（如"破势"、"针女"等）。

```
"御魂类型": "破势"
```

**作用**: 用于标识当前配置针对的御魂类型，方便管理多个配置文件。

#### 1\. stage\_thresholds - 阶段概率阈值

定义了在不同强化等级时，继续强化所需的最低达标概率。

```
"stage_thresholds": {
  "3": 0.30,   // +0 -> +3 需要至少30%的概率达到Top N
  "6": 0.40,   // +3 -> +6 需要至少40%的概率
  "9": 0.50,   // +6 -> +9 需要至少50%的概率
  "12": 0.60   // +9 -> +12 需要至少60%的概率
}
```

**作用**: 控制决策的保守程度。阈值越高，系统越保守，会弃置更多御魂。

#### 2\. top\_n\_values - Top N阈值

定义了不同类型御魂的排名目标。**极限属性已拆分为独立配置**。

```
"top_n_values": {
  "极限速度": 3,    // 极限速度御魂目标是进入Top 3
  "极限生命": 3,    // 极限生命御魂目标是进入Top 3
  "极限攻击": 3,    // 极限攻击御魂目标是进入Top 3
  "极限防御": 3,    // 极限防御御魂目标是进入Top 3
  "极限命中": 3,    // 极限命中御魂目标是进入Top 3
  "极限抵抗": 3,    // 极限抵抗御魂目标是进入Top 3
  "输出御魂": 10,   // 输出御魂目标是进入Top 10
  "奶妈御魂": 10,   // 奶妈御魂目标是进入Top 10
  "防御输出": 10,   // 防御输出御魂目标是进入Top 10
  "其他": 5         // 其他类型御魂目标是进入Top 5
}
```

**作用**: 控制每种类型御魂的保留标准。Top N越小，标准越严格。

**重要变化**:

*   原来的 `"极限属性": 3` 已拆分为6个独立配置
*   每种极限属性可以单独设置不同的Top N值
*   例如：可以设置 `"极限速度": 1`（只保留最快的），`"极限生命": 5`（保留前5名）

#### 3\. extreme\_attr\_mapping - 极限属性映射

定义了极限属性对应的副属性类型。

```
"extreme_attr_mapping": {
  "速度": ["速度"],
  "生命": ["生命加成"],      // 注意：只看加成，不看数值
  "攻击": ["攻击加成"],      // 注意：只看加成，不看数值
  "防御": ["防御加成"],      // 注意：只看加成，不看数值
  "命中": ["效果命中"],
  "抵抗": ["效果抵抗"]
}
```

**作用**:

*   当配置中有"极限生命"时，系统会查找御魂上的"生命加成"副属性
*   **重要**: 生命/攻击/防御的极限属性只看百分比加成，不看固定数值
    *   极限生命 → 只看生命加成%，不看生命数值
    *   极限攻击 → 只看攻击加成%，不看攻击数值
    *   极限防御 → 只看防御加成%，不看防御数值

#### 4\. extreme\_ignore\_main\_attr - 极限属性是否忽略主属性

定义了极限属性在排名对比时是否忽略主属性。

```
"extreme_ignore_main_attr": {
  "速度": true,     // 极限速度可以跨主属性对比
  "生命": false,    // 极限生命必须主属性为生命加成
  "攻击": false,    // 极限攻击必须主属性为攻击加成
  "防御": false,    // 极限防御必须主属性为防御加成
  "命中": true,     // 极限命中可以跨主属性对比
  "抵抗": true      // 极限抵抗可以跨主属性对比
}
```

**作用**:

*   `true`: 排名时忽略主属性，只按位置和套装筛选（如极限速度可以和所有主属性的御魂对比）
*   `false`: 排名时必须主属性一致（如极限生命只和生命加成主属性的御魂对比）

**原因**:

*   极限生命/攻击/防御：主属性必须与极限类型一致才有意义
*   极限速度/命中/抵抗：主属性可以是任意的，应该跨主属性对比找最好的

#### 5\. base\_compatible\_attrs - 基础适配属性集

定义了不同类型御魂的基础适配副属性。

```
"base_compatible_attrs": {
  "输出御魂": ["暴击", "暴击伤害", "速度", "攻击加成"],
  "奶妈御魂": ["暴击", "暴击伤害", "速度", "生命加成"],
  "防御输出": ["暴击", "暴击伤害", "速度", "防御加成"],
  "其他": ["速度", "暴击", "暴击伤害"]
}
```

**作用**: 定义评分时哪些副属性会被计入。不在此列表中的副属性评分为0。

#### 6\. two\_set\_mapping - 两件套属性映射

定义了极限属性与两件套效果的对应关系，用于判断是否可以跨套装对比。

```
"two_set_mapping": {
  "生命": "生命加成",   // 极限生命对应的两件套效果是"生命加成"
  "攻击": "攻击",       // 极限攻击对应的两件套效果是"攻击"
  "防御": "防御加成"    // 极限防御对应的两件套效果是"防御加成"
}
```

**作用**:

*   用于极限属性御魂的跨套装对比
*   例如：当评估一个"极限生命"的御魂时，如果该御魂的两件套效果是"生命加成"，则可以与所有两件套效果为"生命加成"的御魂（如破势、地藏像、蚌精等）进行排名对比
*   这样可以找到同类型中最好的御魂，而不局限于单一套装

**示例**:

*   破势套装（两件套：生命加成10%）的极限生命御魂，可以和地藏像、蚌精等所有"生命加成"两件套的御魂一起排名
*   针女套装（两件套：攻击15）的极限攻击御魂，可以和所有"攻击"两件套的御魂一起排名

### 修改建议

**调整保守程度**: 修改 `stage_thresholds`

*   想要更激进（保留更多御魂）→ 降低阈值（如改为0.20, 0.30, 0.40, 0.50）
*   想要更保守（只保留最好的）→ 提高阈值（如改为0.40, 0.50, 0.60, 0.70）

**调整排名目标**: 修改 `top_n_values`

*   想要更严格的标准 → 减小Top N值
*   想要更宽松的标准 → 增大Top N值
*   **新功能**: 可以为每种极限属性单独设置不同的Top N值

**添加新的极限属性**: 在 `extreme_attr_mapping` 中添加新条目

**修改评分权重**: 在各御魂配置文件中修改副属性权重

### 注意事项

*   修改配置后无需重启，下次运行程序时会自动加载新配置
*   JSON格式要求严格，注意逗号和引号
*   建议修改前备份原配置文件

```
"极限速度": 1,   // 极限速度只保留第一名
"极限生命": 5,   // 极限生命保留前5名
"极限攻击": 3    // 极限攻击保留前3名
```
