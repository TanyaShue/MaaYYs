# 阴阳师御魂数据结构说明

## 概述

本文档说明 `convert_yyx.py` 脚本转换后的JSON数据结构。

## 输入格式

脚本读取 **yyx格式** 的JSON文件，该格式包含完整的御魂信息，结构如下：

```
{
  "version": "2.0.1",
  "timestamp": "...",
  "data": {
    "hero_equips": [
      {
        "id": "5fcc5080b2a81c0e1d5a743a",
        "suit_id": 300019,
        "kind": "火灵",
        "pos": 5,
        "quality": 6,
        "level": 15,
        "base_attr": {
          "type": "HpRate",
          "value": 0.55
        },
        "random_attrs": [
          {
            "type": "CritPower",
            "value": 0.06620318493184545
          }
        ]
      }
    ]
  }
}
```

## 输出格式

转换后的JSON文件结构：

```
{
  "meta": {
    "description": "阴阳师御魂数据 - 从yyx格式转换",
    "count": 9279,

    {
```

## 字段说明

### meta（元数据）

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| description | string | 数据描述 |
| count | number | 御魂总数 |
| source | string | 数据来源 |

### equips（御魂列表）

每个御魂对象包含以下字段：

| 字段 | 类型 | 说明 | 示例 |
| --- | --- | --- | --- |
| 位置 | number | 御魂位置（0-5） | 5 |
| 类型 | string | 御魂套装名称 | "火灵" |
| 品质 | number | 御魂品质（6=六星） | 6 |
| 等级 | number | 御魂等级（0/3/6/9/12/15） | 15 |
| 主属性 | object | 主属性对象 | 见下方 |
| 副属性 | array | 副属性数组 | 见下方 |

### 主属性对象

| 字段 | 类型 | 说明 | 示例 |
| --- | --- | --- | --- |
| 类型 | string | 属性类型中文名 | "生命加成" |
| 数值 | string | 格式化后的数值 | "55.00%" 或 "486" |

### 副属性对象

| 字段 | 类型 | 说明 | 示例 |
| --- | --- | --- | --- |
| 类型 | string | 属性类型中文名 | "暴击伤害" |
| 数值 | string | 格式化后的数值 | "6.62%" 或 "10.90" |
| 强化次数 | number | 估算的强化次数（1-6） | 2 |
| 等效强化次数 | number | 精确的等效次数 | 1.84 |

**百分比属性**：显示为百分比格式（如 "55.00%"）

**数值属性**：显示为数字（如 "486" 或 "10.90"）

*   攻击、生命、防御
*   速度

## 强化次数计算说明

### 计算方法

副属性的强化次数基于 **平均成长值** 计算：

```
平均成长值 = 最大成长值 × 0.9
强化次数 = round(当前数值 / 平均成长值)
```

### 为什么使用 0.9 系数？

御魂副属性每次强化的数值是随机的：

*   **最小值**：最大成长值 × 0.8
*   **最大值**：最大成长值 × 1.0
*   **平均值**：(0.8 + 1.0) / 2 = **0.9**

使用平均值能更准确地估算实际强化次数。

### 最大成长值表

| 属性 | 最大成长值 |
| --- | --- |
| 暴击伤害 | 4% |
| 暴击 | 3% |
| 速度 | 3 |
| 生命加成 | 3% |
| 攻击加成 | 3% |
| 防御加成 | 3% |
| 效果命中 | 4% |
| 效果抵抗 | 4% |
| 攻击 | 27 |
| 防御 | 5 |
| 生命 | 114 |

### 示例

暴击伤害副属性值为 6.62%：

*   平均成长值 = 4% × 0.9 = 3.6%
*   等效强化次数 = 6.62% / 3.6% = 1.84
*   强化次数 = round(1.84) = **2次**

## 使用脚本

### 命令格式

```
python convert_yyx.py -i <输入文件> [-o <输出文件>]
```

### 参数说明

| 参数 | 必需 | 说明 |
| --- | --- | --- |
| \-i, --input | 是 | 输入的yyx格式JSON文件路径 |
| \-o, --output | 否 | 输出文件路径（默认为输入文件同目录下的 yys\_yuhun\_from\_yyx.json） |

### 使用示例

```
# 使用默认输出路径
python convert_yyx.py -i "yyx-26-05-15_03-22-20.json"

# 指定输出路径
python convert_yyx.py -i "input.json" -o "output.json"
```

## 注意事项

1.  **位置编号**：游戏中的位置是1-6号位，但数据中使用0-5表示
2.  **强化次数范围**：1-6次（1次为初始，最多强化5次）
3.  **等效强化次数**：保留2位小数，更精确地反映副属性质量
4.  **数值精度**：百分比保留2位小数，其他数值根据情况保留整数或小数
